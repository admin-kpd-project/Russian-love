# Эволюция чата (realtime)

1. **Сейчас (MVP):** полный функционал через REST из OpenAPI — список бесед, история с пагинацией, отправка, мягкое удаление, загрузка медиа через presigned URL.
2. **Следующий этап:** WebSocket в том же FastAPI-модуле; при нескольких репликах API — Redis pub/sub для доставки событий между процессами.
3. **При высокой нагрузке:** выделенный chat-layer (отдельный сервис), монолит остаётся источником правды по истории в PostgreSQL.
